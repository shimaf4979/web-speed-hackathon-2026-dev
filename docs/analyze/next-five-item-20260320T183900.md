## 4. 実装前の仮説

### CLS の仮説
- `isLoadingActiveUser` が true の間、全ページレイアウト（Navigation + main）がレンダリングされず、auth 完了時に一気に表示 → 大きな CLS
- MovieArea が `<Suspense fallback={null}>` で lazy ロードされるため、チャンクロード時に動画エリアが 0→正方形に膨張 → 下の記事が押し下げ
- アバター画像に CSS サイズ指定がなく「Unsized image element」としてCLS発生

### LCP の仮説
- LCP 画像の Load Delay (45%) は、JS → React → API → render のチェーンが長いため
- Render Delay (47%) は、React がまだ `<img>` を DOM に追加していないため
- auth ブロック解除 + preload + prefetch で大幅に短縮可能

## 5. 採用した方針

**安全優先案**: レギュレーション・VRT に影響しない範囲で、レンダリングパイプラインの無駄なブロッキングを除去。

1. auth チェック完了を待たずに AppPage 全体を即時レンダリング
2. MovieArea/SoundArea の Suspense fallback にスペース確保用プレースホルダー
3. アバター画像に `w-full h-full object-cover` で明示的サイズ
4. `<link rel="preload">` で LCP 画像を早期ダウンロード開始
5. インラインスクリプトでタイムライン API を即時 prefetch
6. index.html に app shell スケルトンを配置して初回ペイントを安定化

### 試みたが採用しなかった方針
- `splitChunks: "all"` + `inject: true` による initial chunk 分割
  - 理由: simulated throttling で追加 HTTP リクエストのオーバーヘッドが大きく、LCP が 4.9s → 7.2s に悪化

## 6. 実装内容

### AppContainer auth ブロック解除
```diff
- if (isLoadingActiveUser) {
-   return (<HelmetProvider><Helmet><title>読込中 - CaX</title></Helmet></HelmetProvider>);
- }
```
`isLoadingActiveUser` 状態変数と関連ロジックを削除。activeUser は null → API 応答後に更新。

### AvatarImage CSS 修正
```diff
- className={className}
+ className={`h-full w-full object-cover${className ? ` ${className}` : ""}`}
```

### TimelineItem Suspense fallback
```tsx
// MovieArea: aspect-ratio 1:1 のプレースホルダー
<Suspense fallback={
  <div className="border-cax-border bg-cax-surface-subtle overflow-hidden rounded-lg border" style={{ aspectRatio: "1/1" }} />
}>
  <MovieArea movie={post.movie} />
</Suspense>

// SoundArea: min-height 100px のプレースホルダー
<Suspense fallback={
  <div className="border-cax-border bg-cax-surface-subtle overflow-hidden rounded-lg border" style={{ minHeight: "100px" }} />
}>
  <SoundArea sound={post.sound} />
</Suspense>
```

### index.html
- `<link rel="preload" as="image" type="image/avif" href="...thumb.avif" fetchpriority="high" />`
- インラインスクリプトで `window.__PREFETCH_TIMELINE__` に fetch promise を格納
- `<div id="app">` 内に Navigation + main のスケルトン HTML

### useInfiniteFetch prefetch 対応
- `prefetchKey` オプション追加
- offset=0 の初回フェッチ時に `window.__PREFETCH_TIMELINE__` を消費

## 9. リスクと未確認事項

- **VRT 正式実行未完了**: Playwright テストがタイムアウトした（手動確認は通過）
- **他ページへの影響**: auth ブロック解除は全ページに影響。Navigation の条件付き表示（DM, Crok 等のログイン必須項目）は activeUser が null → 更新後に表示が切り替わるため、ログイン済みユーザーで一瞬表示が異なる可能性
- **prefetch は home のみ**: タイムライン API の prefetch はホームのみ。他ページは通常の fetch
- **LCP 画像 ID はハードコード**: seed data 変更時に preload URL の更新が必要
