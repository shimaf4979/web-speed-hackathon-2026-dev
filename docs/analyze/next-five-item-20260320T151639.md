# Next Five Item Report: Replace `moment` With `Intl`

## 1. Target item

- Replace `moment` in the client with `Intl.DateTimeFormat` / `Intl.RelativeTimeFormat`.

## 3. Impact scope

- Home timeline: `application/client/src/components/timeline/TimelineItem.tsx`
- Post detail: `application/client/src/components/post/PostItem.tsx`
- Comment display: `application/client/src/components/post/CommentItem.tsx`

- DM detail: `application/client/src/components/direct_message/DirectMessagePage.tsx`
- DM list: `application/client/src/components/direct_message/DirectMessageListPage.tsx`

## 4. Pre-change hypothesis

- `moment` was only used for Japanese date rendering and relative time text.
- Removing it should reduce shipped JavaScript without changing core behavior.
- The main risk is text drift around `fromNow()` output on the DM list.

## 5. Chosen approach

- Add `application/client/src/utils/date.ts` with small shared format helpers.
- Use `Intl.DateTimeFormat` for `LL`-style date and `HH:mm` time.
- Use `Intl.RelativeTimeFormat` with Moment-like thresholds for Japanese relative time.
- Remove the direct `moment` dependency from `application/client/package.json`.

## 6. Implementation summary

- Added `formatJapaneseDate`, `formatJapaneseTime`, `formatRelativeTimeJa`, and `toIsoDateTime`.
- Replaced all direct `moment(...)` calls in the six target components.
- Updated the workspace lockfile after removing the direct dependency.
